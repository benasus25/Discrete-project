Theory

Inorder is one of the depth-first search traversal algorithms in a binary tree where the left child is traversed first followed by the root node followed by the right child.

Pseudo-Code
inorder-tree-walk(x){where x is the root of the tree}

  1. If x is not equal to NIL
  2. inorder-tree-walk(x.left)
  3. print(x.data)
  4. inorder-tree-walk(x.right)

The time complexity of the inorder-tree-walk in an n-node tree is ùõ©(n)

Proof of time complexity-

  • Since the algorithm traverses every node, so T(n) = gamma(n) --(i)
  • The algorithm takes constant time on an empty subtree, so T(0) = c
  • Now consider a tree with n>0, if the algorithm is called on the node x, let the number of nodes in the left subtree is k, and the number of nodes in the right subtree is n-k-1 The T(n) <= T(k) + T(n-k-1) + d for some constant d > 0 and it reflects the upper bound on T(n), exclusive of the recursive calls.
  • According to the substitution method, T(n) = O(n), if T(n) <= (c+d)n+c
  • Now, for n = 0, (c+d) * 0 + c = c = T(0)
  • For n > 0,
    T(n) <= T(k) + T(n-k-1) + d
    = ((c+d)*k + c) + ((c+d)*(n-k-1) + c) + d
    = (c+d)*n + c
  • Hence, T(n) = O(n) --(ii)
  • From (i) and (ii), T(n) = ùõ©(n)
